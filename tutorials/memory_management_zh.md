# Gemini Claw 记忆管理

为了能够提供持久的、能够自我演进的交互体验，Gemini Claw 提供了一套基于本地文件的长期记忆系统。这使得机器人在跨会话、跨任务甚至跨天数的对话中依然能够保留上下文和进展情况。

---

## 1. 基于本地文件的记忆存储

机器人的记忆完全由本地的 Markdown 文件支持，这保证了极高的耐用性与透明度。以下是存储不同数据的核心文件：

- **`memory/Progress.md`**：存放长期项目的当前进展与待办事项。
- **`memory/Learnings.md`**：记录关于用户偏好、代码结构、架构模式以及相关风格的最佳实践。
- **`memory/Errors.md`**：跟踪发生的重要错误、排查解决步骤以及避免重复同样错误的经验等。

由于它们全部是纯文本的 Markdown 文件，你可以随时用任何文本编辑器来审视、修正或者直接编辑机器人的记忆。

---

## 2. 自动化记忆整合机制

长时间积累的日志和频繁更新的数据很容易变得冗长，因此必须定期进行自动化整合。

- **日志更新机制：** 带有新进展和新见解的详细内容会定期自动附加进日志文件或草稿文件。
- **定时任务（Cronjobs）：** 系统设有专门用于维护记忆的定时任务配置（位于 `cronjobs/Memory Updates.md`）。该自动化流程在后台运行每天的记忆去重和提炼工作：
  - 扫描最新的记忆日志变动。
  - 提取关键的新学习点、核心进展，并剔除已经过时的或冗余的信息。
  - 将精炼后的结果更新写回到核心记忆文件。
