# Gemini Claw 后台任务

在执行耗时较长或阻塞型的操作（如训练模型、本地构建、批量数据整理等）时，使用同步消息等待可能会导致超时或资源占用。Gemini Claw 允许安全地脱离主线程运行持续的后台任务。

---

## 1. 使用 `nohup` 运行脱离任务

任何长时间运行的操作都必须脱离终端会话独立执行。
- **运行机制：** 使用 `nohup <command> > output.log 2>&1 &` 来发起后台进程。
- **日志记录：** 必须始终将标准输出和错误输出重定向到指定的日志文件，以供智能体和心跳监控机制追踪并读取进展。

---

## 2. 进程与生命周期追踪

一旦启动后台任务，智能体必须将其元数据记录到状态文件中，以便跟踪其运行。
- **状态追踪：** 打开 `memory/Background Tasks.md` 记录对应的进程 ID (PID)、启动时间、相关的日志路径以及预定超时的截止时间。
- **Discord 频道关联：** 必须同样记录触发任务的对应 Discord 频道 ID `channel_id`，以确保在任务完成时能够准确向对应的频道或线程发送后续结果报告。

---

## 3. 通过心跳机制 (`Heartbeat`) 进行监控与报告

系统利用专门的定时心跳任务来进行后台进程的管理与状态汇报。
- **自动化检测：** 位于 `cronjobs/Heartbeat.md` 中的心跳任务会自动定期运行。
- **心跳流程：** 
  - 轮询 `memory/Background Tasks.md` 中记录的所有活跃 PID 及其状态。
  - 检查正在运行的任务是否已满足完成条件（成功退出或达到失败超时状态）。
  - 使用形如 `[to_channel: {channel_id}]` 的消息路由标签，直接向对应的 Discord 频道原生推送最终的状态报告及日志内容。
